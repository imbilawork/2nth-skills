01

OVERVIEW

Open Design (OD) is a web app + local Node daemon that turns any installed coding-agent CLI into a design engine for prototypes, decks, dashboards, mobile screens, marketing pages, and document templates. The daemon auto-detects whichever supported CLI is on PATH and routes design work through it. Users with no CLI fall back to a bundled OpenAI-compatible BYOK proxy.

Anthropic's Claude Design launched 2026-04-17 (Opus 4.7). Open Design followed eleven days later, on 2026-04-28. It's young — but architecturally complete and end-to-end working. This skill packages the verified facts, install patterns, SA-context analysis, and an honest vs-Claude-Design comparison so your AI agent can guide adoption decisions without propagating the inflated numbers circulating on social media.

HOW IT WORKS

OD ships zero models. The daemon is the orchestrator; whatever coding-agent CLI you already have on PATH becomes the engine. Skills and Design Systems are prompt-stack injections, not separate models.

All steps local. Only the model API call leaves the device — and only if you're not running a local model.

CLI auto-detect priority

The daemon picks the first match on PATH in this order:

  1. Claude Code
  2. Codex CLI
  3. Cursor Agent
  4. Gemini CLI
  5. OpenCode
  6. Qwen Code
  7. GitHub Copilot CLI

If none are found, the daemon enters BYOK proxy mode and the UI surfaces an OpenAI-compatible config form (baseUrl, apiKey, model). Force a specific adapter with OD_FORCE_AGENT=codex pnpm dev.

Quickstart

git clone https://github.com/nexu-io/open-design
cd open-design
pnpm install && pnpm dev

UI opens at http://localhost:5173. The daemon binds to a local socket under /tmp/open-design/ipc/<namespace>/<app>.sock.

05

5-DIMENSIONAL SELF-CRITIQUE

Before emitting an artifact, the agent runs a 5-dimensional self-check sourced from alchaincyf/huashu-design. The full prompt lives at src/prompts/discovery.ts in the OD repo.

Two things worth knowing:

  1. The dimensions are real — but the exact labels and numeric threshold popularly cited on LinkedIn ("philosophy / hierarchy / execution / specificity / restraint, scores below 3/5 regenerate") are not yet verified against the source. Fetch src/prompts/discovery.ts directly before quoting them.
  2. It's a prompt-stack rubric, not a separate model. Performance depends entirely on the underlying CLI's instruction-following. Strong with Opus 4.7 / Sonnet 4.6 / Codex; weaker with smaller open models.
Don't promise regeneration semantics

Because the critique is a prompt-stack rubric and not a deterministic engine, don't promise customers a specific regeneration threshold. Behaviour shifts when the underlying model changes.

06

SOUTH AFRICAN CONTEXT

OD is materially better-fitted to South African design workflows than Claude Design on three axes: data protection (POPIA), cost (ZAR/USD FX exposure), and operational continuity (load-shedding).

POPIA mapping

  • Condition 4 — Further Processing Limitation (s15). Claude Design routes every brief through Anthropic's US infrastructure. OD with a local CLI processes briefs under your existing CLI terms; OD with local Ollama keeps data on the device entirely.
  • Condition 7 — Security Safeguards (s19). OD's daemon edge blocks SSRF to loopback / link-local / RFC1918, stores credentials at file mode 0600, and isolates rendered artifacts in a sandboxed srcdoc iframe.
  • Cross-border transfers (s72). Claude Design = automatic transfer to the US. OD with local Ollama = no transfer. OD with a US BYOK provider = same transfer concern as Claude Design.

Load-shedding playbook

Stage-6 evening + congested international peer = no Claude Design. OD with local Ollama (e.g. qwen2.5-coder:32b on a 32GB MacBook) requires zero internet — a charged laptop and 4 hours of stage 6 still gets a full design session done.

07

VS CLAUDE DESIGN

Both are first-draft tools. Neither replaces a senior designer. Pick based on workflow fit, not feature counts.

Pick…If you…
Claude DesignNeed Canva export, have non-technical collaborators, live in a slide-deck-to-Canva pipeline, want web capture from a live site, or value design-system auto-build from your codebase.
Open DesignLive in your CLI, don't want a second subscription, have POPIA / data-sovereignty constraints, want the artifact to land directly in your code workspace, or need offline-capable design work.

Hybrid pattern most SA teams actually want

Use Claude Design for the first 30 minutes of stakeholder exploration (sliders are genuinely faster for "what if it were warmer" conversations). Then export to HTML, drop into the Open Design workspace, and iterate to ship from there. R380/mo for the design lead's Claude Pro seat; R0 for everyone else's OD usage.
