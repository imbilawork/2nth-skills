WHAT IS TWENTY

Twenty is an open-source CRM built as a modern alternative to Salesforce. It's GraphQL-first, written in TypeScript (NestJS backend, React frontend), and stores data in PostgreSQL. The entire data model is customisable — you can add custom objects and fields without touching code.

DATA MODEL

Twenty's core objects map to standard CRM concepts. Every object supports custom fields. The Metadata API lets agents inspect and extend the schema programmatically.

ObjectDescriptionKey FieldsRelationships
peopleIndividual contactsname, email, phone, city, jobTitle→ companies, opportunities, notes
companiesOrganisations / accountsname, domainName, employees, annualRecurringRevenue→ people, opportunities
opportunitiesSales pipeline dealsname, amount, stage, closeDate, probability→ companies, people, notes
notesActivity notestitle, body, createdAt→ people, companies, opportunities

workspaceMemberTeam membersname, email, role→ tasks, notes

Import Order

When seeding or migrating data, follow this sequence to satisfy foreign key constraints:

# Required import sequence
1. Companies
2. People        # references companies
3. Opportunities # references companies + people
4. Notes / Tasks # reference any of the above
5. Custom Objects

AUTHENTICATION

All Twenty APIs use Bearer token auth. Generate API keys in Settings → APIs & Webhooks. Keys are workspace-scoped and can have roles assigned to limit permissions.

# All requests require this header
Authorization: Bearer YOUR_API_KEY

# Cloud endpoint
https://api.twenty.com/

# Self-hosted endpoint
https://your-twenty-instance.com/
Security note: Copy your API key immediately after creation — Twenty does not show it again. Create separate keys per integration and assign minimum required permissions. Disable compromised keys immediately from the same settings page.

Webhook Signature Verification

Incoming webhooks include two headers for verification:

X-Twenty-Webhook-Signature: hmac-sha256-signature
X-Twenty-Webhook-Timestamp: 1712000000
// Verify in a Cloudflare Worker
async function verifyTwentyWebhook(request: Request, secret: string): Promise<boolean> {
  const sig = request.headers.get('X-Twenty-Webhook-Signature');
  const ts  = request.headers.get('X-Twenty-Webhook-Timestamp');
  const body = await request.text();

  const key = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(secret),
    { name: 'HMAC', hash: 'SHA-256' }, false, ['sign']
  );
  const expected = await crypto.subtle.sign(
    'HMAC', key, new TextEncoder().encode(`${ts}.${body}`)
  );
  return btoa(String.fromCharCode(...new Uint8Array(expected))) === sig;
}

GRAPHQL API

The GraphQL API is Twenty's primary integration surface. It supports relationship traversal, filtering, sorting, pagination, and batch upserts — all in a single request. Rate limit: 100 req/min, batch up to 60 records per call.

Query People with Company

query GetPeopleWithCompany {
  people(
    filter: { city: { eq: "Cape Town" } }
    orderBy: { createdAt: DescNullsLast }
    first: 20
  ) {
    edges {
      node {
        id
        name { firstName lastName }
        email { primaryEmail }
        jobTitle
        company {
          name
          domainName { primaryLinkUrl }
          annualRecurringRevenue { amountMicros currencyCode }
        }
      }
    }
    pageInfo { hasNextPage endCursor }
  }
}

Create an Opportunity

mutation CreateOpportunity {
  createOpportunity(data: {
    name: "Acme Corp — Phase 1"
    amount: { amountMicros: 500000000, currencyCode: "ZAR" }
    stage: "MEETING_SCHEDULED"
    closeDate: "2026-06-30"
    companyId: "company-uuid-here"
  }) {
    id name stage
    amount { amountMicros currencyCode }
  }
}

Batch Upsert People

mutation UpsertPeople {
  upsertPeople(
    data: [
      { name: { firstName: "Guy", lastName: "Smith" }, email: { primaryEmail: "guy@acme.com" } },
      { name: { firstName: "Rowan", lastName: "Jones" }, email: { primaryEmail: "rowan@acme.com" } }

REST API

The REST API mirrors the GraphQL API for standard CRUD. Use it when your agent stack doesn't support GraphQL, or for simple integrations. Endpoint: /rest/{objectName}.

MethodEndpointAction
GET/rest/peopleList all people (supports filter, orderBy, limit, offset)
GET/rest/people/:idGet a single person by ID
POST/rest/peopleCreate a person
PATCH/rest/people/:idUpdate a person
DELETE/rest/people/:idDelete a person
GET/rest/metadata/objectsList all object types (standard + custom)
GET/rest/metadata/fieldsList all fields for an object
 
# List open opportunities, sorted by amount
curl https://api.twenty.com/rest/opportunities \
  -H "Authorization: Bearer YOUR_KEY" \
  -G \
  --data-urlencode 'filter=stage[eq]:MEETING_SCHEDULED' \
  --data-urlencode 'orderBy=amount[amountMicros][DescNullsLast]' \
  --data-urlencode 'limit=50'

WEBHOOKS

Twenty pushes real-time events to your endpoint when records change. Configure in Settings → APIs & Webhooks → Webhooks. Your endpoint must respond with HTTP 2xx.

Event Types

Cloudflare Worker Webhook Handler

export default {
  async fetch(req: Request, env: Env) {
    if (req.method !== 'POST') return new Response('Method not allowed', { status: 405 });

    const body = await req.text();
    const payload = JSON.parse(body);

    switch (payload.event) {
      case 'opportunity.updated':
        if (payload.data.stage === 'CLOSED_WON') {
          await triggerWinWorkflow(payload.data, env);
        }
        break;
      case 'person.created':
        await enrichNewContact(payload.data, env);
        break;
    }

    return new Response('ok');
  }
};

MCP SERVER

Twenty has a community MCP server that exposes CRM data as tools Claude (and other MCP-compatible agents) can call directly. The agent can query contacts, create opportunities, add notes, and search the pipeline — all in natural language, no code required.

What this means for operators: You can give Claude access to a client's Twenty instance and ask it to "find all open deals over R500k in Cape Town and draft a follow-up email for each." Claude calls the MCP tools, gets the data, and produces the output — no custom integration code needed.

Setup in Claude Code

# Add to .claude/settings.json or claude_desktop_config.json
{
  "mcpServers": {
    "twenty": {
      "command": "npx",
      "args": ["-y", "twenty-mcp-server"],
      "env": {
        "TWENTY_API_URL": "https://api.twenty.com",
        "TWENTY_API_KEY": "your-api-key"
      }
    }
  }
}

AGENT PATTERNS

Six production patterns for AI agents working with Twenty CRM.

Pattern 1 — Inbound Lead Enrichment

A webhook fires when a new person is created. A Worker calls Workers AI to enrich the record — infer company size, industry, and likely role from the email domain and any available context — then patches the record back.

// Worker: enrich new contact via Workers AI
async function enrichNewContact(person: TwentyPerson, env: Env) {
  const prompt = `Analyse this contact and return JSON:
  Email: ${person.email?.primaryEmail}
  Company: ${person.company?.name}
  Return: { industry, companySize, seniorityLevel, likelyBudget }`;

  const ai = await env.AI.run('@cf/meta/llama-3.3-70b-instruct-fp8-fast', {
    messages: [{ role: 'user', content: prompt }]
  });

  const enriched = JSON.parse(ai.response);

  // Patch back to Twenty via GraphQL
  await twentyMutation(`
    mutation { updatePerson(id: "${person.id}", data: {
      city: "${enriched.industry}"
    }) { id } }
  `, env.TWENTY_API_KEY);
}

Pattern 2 — Pipeline Stage Alert

When an opportunity moves to a key stage, fire a Slack message or email with AI-generated context about the deal and suggested next actions.

case 'opportunity.updated': {

Pattern 3 — Natural Language CRM Query

Accept a plain-English question, convert it to a GraphQL query via an LLM, execute against Twenty, and return a formatted answer.

async function nlQuery(question: string, env: Env) {
  // Step 1: Convert NL to GraphQL
  const gql = await env.AI.run('@cf/meta/llama-3.3-70b-instruct-fp8-fast', {
    messages: [{
      role: 'system',
      content: 'Convert the user question to a Twenty CRM GraphQL query. Return only valid GraphQL.'
    }, {
      role: 'user', content: question
    }]
  });

  // Step 2: Execute the query
  const data = await twentyQuery(gql.response, env.TWENTY_API_KEY);

  // Step 3: Summarise the result
  const answer = await env.AI.run('@cf/meta/llama-3.3-70b-instruct-fp8-fast', {
    messages: [{
      role: 'user',
      content: `Question: ${question}\nData: ${JSON.stringify(data)}\nAnswer in plain English.`
    }]
  });

  return answer.response;
}

Pattern 4 — Bulk Import from External Source

Pull contacts from an external source (CSV, ERP, spreadsheet) and upsert into Twenty using batch mutations. Respects the import order and rate limits.

async function batchImportPeople(contacts: Contact[], apiKey: string) {
  // Twenty allows 60 records per batch call
  for (let i = 0; i < contacts.length; i += 60) {
    const chunk = contacts.slice(i, i + 60);
    await twentyMutation(`
      mutation {
        upsertPeople(
          data: [${chunk.map(c => `{
            name: { firstName: "${c.firstName}", lastName: "${c.lastName}" },
            email: { primaryEmail: "${c.email}" }
          }`).join(',')}]
          uniqueOn: [EMAIL_PRIMARY_EMAIL]
        ) { id }
      }
    `, apiKey);
    // Respect rate limit: 100 req/min
    await new Promise(r => setTimeout(r, 700));
  }
}

Helper: Twenty GraphQL client for Workers

// src/twenty.ts — reusable in any Worker
export async function twentyQuery(query: string, apiKey: string, variables?: object) {
  const res = await fetch('https://api.twenty.com/graphql', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${apiKey}`,
    },
    body: JSON.stringify({ query, variables }),
  });
  const json = await res.json() as { data: any; errors?: any[] };
  if (json.errors?.length) throw new Error(json.errors[0].message);
  return json.data;
}

export const twentyMutation = twentyQuery; // same function, different name for clarity

SETUP GUIDE

Get a Cloudflare Worker talking to Twenty in under 10 minutes.

01

Get your API key

In Twenty: Settings → APIs & Webhooks → API Keys → Generate. Copy it immediately.

02

Store as a Worker secret

npx wrangler secret put TWENTY_API_KEY
03

Add the helper to your Worker

Copy the twentyQuery helper above into src/twenty.ts in your Worker project.

04

Test a query

const data = await twentyQuery(`
  query { people(first: 5) {
    edges { node { id name { firstName } } }
  }}
`, env.TWENTY_API_KEY);
05

Register a webhook

In Twenty: Settings → APIs & Webhooks → Webhooks → Add. Set your Worker URL as the endpoint. Select the events you need.

06

For Claude / MCP — add the MCP server config

Add the Twenty MCP server to your .claude/settings.json as shown in section 07.
